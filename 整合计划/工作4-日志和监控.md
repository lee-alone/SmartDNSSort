# 工作4：日志和监控

## 目标

实现分类日志输出、监控接口和 Dashboard 显示功能。

## 任务清单

### 4.1 实现分类日志输出
- [ ] 定义日志标签：[INIT]、[FWD]、[REC]、[NET]
- [ ] 修改日志输出函数支持标签
- [ ] 实现日志级别控制（DEBUG、INFO、WARN、ERROR）
- [ ] 实现日志格式统一

### 4.2 实现递归模块日志
- [ ] 启动日志：`[REC] Loading root hints from named.cache...`
- [ ] 解析日志：`[REC] Parsed 13 root servers (IPv4/IPv6)`
- [ ] 查询日志：`[REC] Iterating: . → com → example.com`
- [ ] 响应日志：`[REC] Response: 93.184.216.34 (TTL: 86400)`
- [ ] 错误日志：`[REC] ERROR: All root hints unreachable`
- [ ] 缓存日志：`[REC] Cache hit for example.com`

### 4.3 实现转发器日志
- [ ] 查询日志：`[FWD] Query: example.com`
- [ ] 上游选择日志：`[FWD] → Trying upstream: 127.0.0.1:5335 (Local Recursive)`
- [ ] 响应日志：`[FWD] ← Got response from 127.0.0.1:5335`
- [ ] 故障日志：`[FWD] ← Upstream 127.0.0.1:5335 (Local Recursive) FAILED`
- [ ] 标记日志：`[FWD] Marking "Local Recursive" as unhealthy`
- [ ] 切换日志：`[FWD] → Trying upstream: 8.8.8.8:53 (External)`

## 日志示例

### 启动日志
```
[INIT] Loading config...
[INIT] Validating upstreams...
[INIT] Found local_recursive upstream: 127.0.0.1:5335
[INIT] Checking if recursive_enabled=true... YES
[INIT] Starting recursive resolver on 127.0.0.1:5335...
[REC] Loading root hints from named.cache...
[REC] Parsed 13 root servers (IPv4/IPv6)
[REC] Recursive resolver started successfully
[INIT] Registering 127.0.0.1:5335 as "Local Recursive" in upstream pool
[INIT] Starting forwarder...
[FWD] Forwarder started with 2 upstreams (1 local, 1 external)
```

### 查询日志
```
[FWD] Query: example.com
[FWD] → Trying upstream: 127.0.0.1:5335 (Local Recursive)
[REC] Iterating: . → com → example.com
[REC] Response: 93.184.216.34 (TTL: 86400)
[FWD] ← Got response from 127.0.0.1:5335
[FWD] Response sent to client
```

### 故障日志
```
[FWD] Query: blocked-domain.com
[FWD] → Trying upstream: 127.0.0.1:5335 (Local Recursive)
[REC] Iterating: . → com → blocked-domain.com
[REC] ERROR: All root hints unreachable (timeout after 5s)
[FWD] ← Upstream 127.0.0.1:5335 (Local Recursive) FAILED
[FWD] Marking "Local Recursive" as unhealthy
[ALERT] Local Recursive resolver is down! Check recursive module status.
[FWD] → Trying upstream: 8.8.8.8:53 (External)
[FWD] ← Got response from 8.8.8.8:53
[FWD] Response sent to client
```
