## 目标

实现递归模块的健康检查、故障感知和自动恢复机制。

- [ ] 监控递归线程的生存状态

### 3.4 实现重启频率限制
- [ ] 设置重启计数器

- [ ] 如果 5 秒内崩溃超过 3 次，放弃重启

### 3.5 实现被动故障检测
- [ ] 在转发器查询失败时标记为故障
- [ ] 连续超时或 Connection Refused 时标记为 Down
- [ ] 自动切换到其他外部 Upstream
- [ ] 记录故障转移日志

- [ ] 恢复成功时标记为 Healthy

- [ ] 记录恢复尝试日志

### 3.7 实现监控接口
- [ ] 提供递归模块状态查询接口
- [ ] 返回运行状态、Uptime、错误信息等
- [ ] 支持 JSON 格式返回

## 故障处理流程

```
递归线程运行
  ↓
定期健康检查
  ↓
检查通过？
  ├─ 是 → 继续运行
  └─ 否 → 标记为故障
       ↓
     尝试重启
       ↓
     重启成功？
       ├─ 是 → 恢复运行
       └─ 否 → 检查重启次数
            ↓
          5秒内超过3次？
            ├─ 是 → 放弃重启，通知主线程
            └─ 否 → 继续尝试重启
```
