# 工作6：named.cache 管理

## 目标

实现 named.cache 文件的可视化管理和更新功能。

## 任务清单

### 6.1 实现文件状态显示
- [ ] 显示 named.cache 版本信息
- [ ] 显示根服务器节点数量（通常 13 个）
- [ ] 显示文件的最后修改时间
- [ ] 显示文件的下载来源
- [ ] 显示文件大小

### 6.2 实现根服务器连通性检查
- [ ] 创建"快速诊断"按钮
- [ ] 实现对 13 个根服务器的连通性检查
- [ ] 显示每个根服务器的状态（✅ 或 ❌）
- [ ] 显示总体连通性（13/13 ✅）
- [ ] 记录检查结果

### 6.3 实现根服务器健康指示灯
- [ ] 为每个根服务器显示健康状态
- [ ] 使用不同颜色表示状态（绿色✅、红色❌）
- [ ] 显示根服务器的 IP 地址
- [ ] 显示根服务器的响应时间
- [ ] 支持点击查看详细信息

### 6.4 实现自动诊断提示
- [ ] 如果大部分根服务器变红，自动提示
- [ ] 提示内容："可能是 UDP 被阻断，建议启用 DoH/DoT"
- [ ] 提供"启用 DoH/DoT"的快捷链接

### 6.5 实现一键更新功能
- [ ] 创建"更新"按钮
- [ ] 实现从官方地址下载最新 named.cache
- [ ] 显示下载进度条
- [ ] 更新前显示"新版本有 X 个根服务器变更"的摘要
- [ ] 更新后自动重新加载

### 6.6 实现更新前验证
- [ ] 检查新版本的有效性
- [ ] 比较新旧版本的差异
- [ ] 显示变更摘要
- [ ] 提供"确认更新"和"取消"选项

### 6.7 实现更新后处理
- [ ] 自动重新加载新的 named.cache
- [ ] 重新初始化根服务器列表
- [ ] 显示更新成功提示
- [ ] 记录更新日志

### 6.8 实现导出和重置功能
- [ ] 实现"导出"功能，导出当前 named.cache
- [ ] 实现"重置"功能，恢复到默认 named.cache
- [ ] 添加确认对话框

## 更新流程

```
用户点击"更新"
  ↓
检查新版本
  ↓
显示变更摘要
  ↓
用户确认
  ↓
下载新版本
  ↓
显示进度条
  ↓
验证新版本
  ↓
备份旧版本
  ↓
替换为新版本
  ↓
重新加载
  ↓
显示成功提示
```
