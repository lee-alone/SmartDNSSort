# 工作5.2 - 快速测试指南

## 🚀 快速启动

### 1. 编译和运行

```bash
# 编译项目
go build -o smartdnssort ./cmd/smartdnssort

# 运行服务
./smartdnssort
```

### 2. 访问 Web 界面

打开浏览器访问：`http://localhost:8080`

### 3. 导航到递归解析器

点击导航栏中的"递归解析器"选项卡

## 🧪 功能测试

### 测试 1：查看状态

**步骤：**
1. 打开递归解析器仪表盘
2. 查看状态卡片中的信息

**预期结果：**
- 显示递归解析器的状态（运行中/已停止）
- 显示运行时间
- 显示监听端口
- 显示查询超时

**验证方式：**
```bash

curl http://localhost:8080/api/resolver/status
```

### 测试 2：控制递归解析器

**步骤：**
1. 点击"启动"按钮
2. 观察状态变化
3. 点击"停止"按钮
4. 观察状态变化
5. 点击"重启"按钮
6. 观察状态变化

**预期结果：**
- 状态徽章颜色变化（绿色/红色）
- 状态文本更新
- 显示成功/失败通知

**验证方式：**
```bash

curl -X POST http://localhost:8080/api/resolver/control \
  -H "Content-Type: application/json" \
  -d '{"action":"start"}'

# 停止递归解析器
curl -X POST http://localhost:8080/api/resolver/control \
  -H "Content-Type: application/json" \
  -d '{"action":"stop"}'

# 重启递归解析器
curl -X POST http://localhost:8080/api/resolver/control \
  -H "Content-Type: application/json" \
  -d '{"action":"restart"}'
```

## 🐛 常见问题

### 问题 1：API 返回 404

**原因：** API 路由未正确注册

**解决方案：**
1. 检查 `webapi/api.go` 中是否注册了路由
2. 检查服务器是否正确启动
3. 查看服务器日志

### 问题 2：前端组件未加载

**原因：** 组件加载器未正确加载 HTML

**解决方案：**
1. 检查浏览器控制台是否有错误
2. 检查 `webapi/web/components/resolver-config.html` 是否存在
3. 检查 `component-loader.js` 中是否添加了组件加载

### 问题 3：状态不更新

**原因：** 定时刷新未启动或 API 返回错误

**解决方案：**
1. 检查浏览器控制台是否有错误
2. 检查 API 是否返回正确的数据
3. 检查 `resolver.js` 中的定时刷新逻辑

### 问题 4：迭代路径跟踪返回空结果

**原因：** TraceResolve 函数未实现

**解决方案：**
1. 实现 `dnsserver.TraceResolve` 函数
2. 返回迭代过程详情
