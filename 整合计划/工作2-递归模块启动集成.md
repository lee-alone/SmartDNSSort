# 工作2：递归模块启动集成

## 目标

在主程序启动流程中集成递归线程启动，实现正确的启动顺序和失败处理。

## 任务清单

### 2.1 设计递归模块启动流程
- [ ] 定义启动顺序：配置初始化 → 启动递归线程 → 启动主转发器
- [ ] 设计递归线程与主线程的通信机制（channel 或 Status Flag）
- [ ] 定义启动失败的处理策略

### 2.2 重构 resolver 的 Transport 层以支持 UDP（关键技术补充）
- [ ] **分析 resolver/transport.go 的当前设计**（目前仅支持 TCP 和 Unix Socket）
- [ ] **为 Transport 接口增加 UDP 支持**
  - 当前 Transport 接口面向流（Stream）：Accept()、Read()、Write()
  - 需要增加 PacketConn 支持，用于 UDP 报文收发
- [ ] **实现 UDPTransport 结构体**
  - 实现 net.PacketConn 接口
  - 支持 UDP 报文的接收和发送
  - 处理 UDP 的无连接特性
- [ ] **修改 resolver/server.go 的处理逻辑**
  - 区分 TCP（连接导向）和 UDP（无连接）的处理方式
  - TCP：保持现有的 Accept 循环 + handleConnection 协程模式
  - UDP：实现 ReadFrom/WriteTo 循环处理报文
- [ ] **添加 UDP 监听器启动逻辑**
  - 在指定端口（5335）启动 UDP 监听
  - 处理 UDP 报文的并发接收
- [ ] 添加错误处理和日志输出

### 2.3 注入全局 Logger（关键技术补充）
- [ ] **在递归模块启动时注入主程序的 Logger 实例**
  - 修改 resolver.NewServer() 函数，接收 Logger 参数
  - 将 Logger 存储在 resolver.Server 结构体中
- [ ] **修改 resolver 包中所有日志输出**
  - 使用注入的 Logger 而不是全局默认 logger
  - 确保所有日志都能通过主程序的日志系统统一管理
- [ ] **配置日志标签**
  - 在 Logger 中配置 `[REC]` 前缀
  - 确保与 `[FWD]` 标签区分

## 启动流程图

```
主程序启动
  ↓
[INIT] 加载配置
  ↓
[INIT] 验证配置
  ↓
[INIT] 启动递归模块
  ↓
[REC] 加载 named.cache
  ↓
[REC] 初始化根服务器
  ↓
[REC] 启动 UDP 监听
  ↓
[REC] 启动 TCP 监听（可选）
  ↓
[REC] 递归模块就绪
  ↓
[INIT] 启动主转发器
  ↓
[FWD] 转发器就绪
  ↓
主程序运行
```

## 错误处理

### 端口被占用
```
[ERROR] Failed to start recursive resolver: port 5335 already in use
[ERROR] Please change the port in recursive.json or stop the conflicting process
[ERROR] Program startup aborted
```

### named.cache 文件不存在
```
[ERROR] Failed to load root hints: named.cache file not found at resolver/named.cache
[ERROR] Please ensure the file exists before starting
[ERROR] Program startup aborted
```

### 根服务器解析失败
```
[ERROR] Failed to parse root hints from named.cache
[ERROR] Error: [具体错误信息]
[ERROR] Program startup aborted
```

### UDP 监听启动失败
```

[ERROR] Error: [具体错误信息]
[ERROR] Program startup aborted
```
